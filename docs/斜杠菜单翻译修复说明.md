# 斜杠菜单翻译修复说明

## 问题描述

在编辑器中输入 "/" 时出现的斜杠命令菜单中，部分项目（如 "Bullet list"、"Numbered list" 等）没有翻译成简体中文。

## 问题原因

虽然 `apps/client/src/features/editor/components/slash-menu/command-list.tsx` 组件已经使用了 `t(item.title)` 和 `t(item.description)` 来翻译菜单项，但简体中文翻译文件 `apps/client/public/locales/zh-CN/translation.json` 中缺少这些翻译键。

## 修复内容

### 1. 添加的翻译键

在 `apps/client/public/locales/zh-CN/translation.json` 中添加了以下翻译：

#### 分类标题
- `"basic": "基础"`

#### 菜单项标题
- `"Text": "文字"`
- `"To-do list": "待办列表"`
- `"Bullet list": "无序列表"`
- `"Numbered list": "有序列表"`
- `"Quote": "引用"`

#### 菜单项描述
- `"Just start typing with plain text.": "只需开始键入纯文本"`
- `"Track tasks with a to-do list.": "使用待办列表跟踪任务"`
- `"Create a simple bullet list.": "创建一个简单的无序列表"`
- `"Create a list with numbering.": "创建一个有序列表"`
- `"Create block quote.": "创建引用块"`

### 2. 清理重复键

在添加翻译的过程中，发现并清理了翻译文件中的重复键，确保每个键只出现一次。

## 相关文件

- `apps/client/public/locales/zh-CN/translation.json` - 简体中文翻译文件
- `apps/client/src/features/editor/components/slash-menu/menu-items.ts` - 斜杠菜单项定义
- `apps/client/src/features/editor/components/slash-menu/command-list.tsx` - 斜杠菜单渲染组件
- `scripts/verify-slash-menu-translation.sh` - 翻译验证脚本

## 验证方法

运行验证脚本：

```bash
bash scripts/verify-slash-menu-translation.sh
```

该脚本会检查所有必需的翻译键是否存在于翻译文件中。

## 测试步骤

1. 启动应用程序
2. 打开任意文档进行编辑
3. 在编辑器中输入 "/"
4. 验证弹出的菜单中所有项目都已正确翻译成简体中文

## 预期结果

- "Bullet list" 显示为 "无序列表"
- "Numbered list" 显示为 "有序列表"
- "To-do list" 显示为 "待办列表"
- 所有其他菜单项也都正确显示简体中文翻译
- 菜单项的描述文本也都正确翻译

## 注意事项

1. 翻译文件使用 JSON 格式，确保语法正确
2. 翻译键必须与 `menu-items.ts` 中的 `title` 和 `description` 字段完全匹配
3. 避免在翻译文件中创建重复的键
