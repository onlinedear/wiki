# 测试斜杠菜单翻译

## 快速测试步骤

1. **启动应用程序**
   ```bash
   # 如果还没有启动，运行：
   pnpm install
   pnpm dev
   ```

2. **打开编辑器**
   - 登录应用
   - 打开或创建一个文档

3. **测试斜杠菜单**
   - 在编辑器中输入 `/`
   - 应该看到一个弹出菜单

4. **验证翻译**
   
   检查以下项目是否正确翻译：

   | 英文原文 | 简体中文翻译 | 状态 |
   |---------|------------|------|
   | Basic | 基础 | ✓ |
   | Text | 文字 | ✓ |
   | To-do list | 待办列表 | ✓ |
   | Heading 1 | 1 级标题 | ✓ |
   | Heading 2 | 2 级标题 | ✓ |
   | Heading 3 | 3 级标题 | ✓ |
   | Bullet list | 无序列表 | ✓ |
   | Numbered list | 有序列表 | ✓ |
   | Quote | 引用 | ✓ |
   | Code | 代码 | ✓ |
   | Divider | 分割线 | ✓ |
   | Image | 图像 | ✓ |
   | Video | 视频 | ✓ |
   | File attachment | 文件附件 | ✓ |
   | Table | 表格 | ✓ |
   | Toggle block | 切换块 | ✓ |
   | Callout | 标注块 | ✓ |

5. **验证描述文本**
   
   检查菜单项下方的描述文本是否也已翻译：

   | 英文描述 | 简体中文翻译 |
   |---------|------------|
   | Just start typing with plain text. | 只需开始键入纯文本 |
   | Track tasks with a to-do list. | 使用待办列表跟踪任务 |
   | Create a simple bullet list. | 创建一个简单的无序列表 |
   | Create a list with numbering. | 创建一个有序列表 |
   | Create block quote. | 创建引用块 |

## 自动验证

运行验证脚本：

```bash
bash scripts/verify-slash-menu-translation.sh
```

## 预期结果

- ✓ 所有菜单项标题都显示简体中文
- ✓ 所有菜单项描述都显示简体中文
- ✓ 分类标题 "Basic" 显示为 "基础"
- ✓ 没有英文文本残留

## 如果发现问题

1. 检查浏览器语言设置是否为简体中文
2. 清除浏览器缓存并刷新页面
3. 检查 `apps/client/public/locales/zh-CN/translation.json` 文件
4. 运行验证脚本确认翻译键存在

## 相关文件

- 翻译文件: `apps/client/public/locales/zh-CN/translation.json`
- 菜单定义: `apps/client/src/features/editor/components/slash-menu/menu-items.ts`
- 菜单组件: `apps/client/src/features/editor/components/slash-menu/command-list.tsx`
