# 页面副本命名优化说明

## 修改概述

根据用户反馈，对页面复制功能的命名进行了两处优化：

### 1. 菜单选项名称优化
- **原名称**: "重复"
- **新名称**: "创建副本"
- **原因**: "创建副本"更清晰地表达了操作的意图

### 2. 副本页面标题格式优化
- **原格式**: `Copy of 标题`
- **新格式**: `标题副本`
- **原因**: 中文习惯将修饰词放在后面，这样更符合中文表达习惯

## 修改文件

### 前端翻译文件
- `apps/client/public/locales/zh-CN/translation.json`
  - `"Duplicate": "创建副本"` (原为 "重复")
  - `"Page duplicated successfully": "副本创建成功"` (原为 "页面复制成功")

### 后端服务文件
- `apps/server/src/core/page/services/page.service.ts`
  - 修改副本命名逻辑：`${originalTitle}副本` (原为 `Copy of ${originalTitle}`)

## 使用示例

### 场景1：复制普通页面
- 原页面标题: "项目文档"
- 副本标题: "项目文档副本"

### 场景2：复制无标题页面
- 原页面标题: "Untitled"
- 副本标题: "Untitled副本"

### 场景3：多次复制
- 第1次复制: "项目文档副本"
- 第2次复制: "项目文档副本副本"
- (用户可以手动重命名以避免重复)

## 验证方法

运行验证脚本：
```bash
bash scripts/verify-duplicate-naming.sh
```

## 注意事项

1. **仅影响中文界面**: 此修改仅更新了中文翻译，其他语言保持不变
2. **仅影响同空间复制**: 副本命名规则仅在同一空间内复制时生效
3. **跨空间复制**: 复制到不同空间时，保持原标题不变
4. **向后兼容**: 不影响已存在的副本页面

## 测试建议

1. 在同一空间内复制页面，验证标题格式为"标题副本"
2. 复制页面到不同空间，验证标题保持不变
3. 验证菜单中显示"创建副本"而非"重复"
4. 验证成功提示消息显示"副本创建成功"
