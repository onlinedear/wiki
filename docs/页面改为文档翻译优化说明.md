# 页面改为文档翻译优化说明

## 需求背景

用户反馈：左侧菜单中的"页面"应该翻译成"文档"，对应的"新建页面"也应该改成"新建文档"，因为"文档"更加贴切。

## 修改原则

### 修改的翻译
将所有指代**文档内容**的"页面"统一改为"文档"，包括：
- 左侧菜单标题
- 新建/创建/编辑操作
- 文档列表和计数
- 文档历史和设置
- 导入/导出操作
- 子文档相关
- 所有提示和错误消息

### 保留的翻译
仅保留指代**网页界面**的"页面"：
- 404 page not found（404页面未找到）
- Sorry, we can't find the page（抱歉，我们无法找到你所需要的页面）

## 注意事项

1. **语义区分**: 
   - "文档" = 用户创建的内容（Document/Page content）
   - "页面" = 网页界面/功能页面（Web page/UI page）

2. **一致性**: 确保整个应用中对"文档"的称呼保持一致

3. **向后兼容**: 不影响现有功能，只是改变显示文本

4. **用户体验**: "文档"更符合中文用户的使用习惯，更容易理解

## 其他语言

此修改仅针对简体中文翻译。其他语言保持原有翻译：
- English: Pages / New page
- 日本語: ページ / 新規ページ
- 한국어: 페이지 / 새 페이지
- 等等...

## 完整性

本次修改已经将所有与文档内容相关的"页面"统一改为"文档"，实现了术语的完全一致性。唯一保留"页面"的地方是404错误页面，因为这里指的是网页本身，而非文档内容。

这样的修改使得整个应用的中文界面更加统一和专业，用户体验更加一致。
