# 页面到文档完整翻译修改总结

## 修改概述

根据用户需求，将中文翻译中所有与文档内容相关的"页面"统一改为"文档"，以提供更一致、更专业的用户体验。

## 修改统计

共修改了 **50+** 处翻译，涵盖以下类别：

### 1. 基础操作（6处）
- Pages → 文档
- New page → 新建文档
- Create page → 创建文档
- page → 个文档
- pages → 个文档
- No pages yet → 暂无文档

### 2. 文档操作（8处）
- Page history → 文档历史
- Export page → 导出文档
- Import pages → 导入文档
- Move page → 移动文档
- Copy page → 复制文档
- Delete page → 删除文档
- Restore page → 恢复文档
- Open page → 打开文档

### 3. 子文档相关（7处）
- Subpages → 子文档
- Include subpages → 包括子文档
- Include sub-pages → 包括子文档
- No subpages → 没有子文档
- Failed to load subpages → 加载子文档失败
- List all subpages of the current page → 列出当前文档的所有子文档
- Restore '{{title}}' and its sub-pages? → 恢复"{{title}}"及其子文档？

### 4. 成功提示（5处）
- Page deleted successfully → 文档已成功删除
- Page moved to trash → 文档已移至垃圾箱
- Page restored successfully → 文档恢复成功
- Page copied successfully → 文档复制成功
- Page duplicated successfully → 副本创建成功

### 5. 错误提示（6处）
- Failed to create page → 创建文档失败
- Failed to delete page → 删除文档失败
- Failed to fetch recent pages → 获取最近文档失败
- Failed to import pages → 导入文档失败
- Failed to load page → 文档加载失败
- Failed to share page → 文档分享失败
- Error fetching page data → 获取文档数据时出错
- Error loading page history → 加载文档历史时出错

### 6. 确认对话框（4处）
- Are you sure you want to delete this page? → 您确定要删除这个文档吗？
- Move this page to trash? → 将此文档移至垃圾箱？
- Permanently delete page? → 永久删除文档？
- Delete this page with all its children and history → 删除这个文档及其子文档和历史记录

### 7. 权限和设置（6处）
- Can create and edit pages in space → 能够在空间中创建和编辑文档
- Can view pages in space but not edit → 能够在空间中读取文档，但不允许编辑
- Has full access to space settings and pages → 能够更改全部空间设置和文档
- Default page edit mode → 默认文档编辑模式
- Choose your preferred page width → 选择您喜欢的文档宽度
- Toggle full page width → 切换全文档宽度

### 8. 分享和公开（5处）
- Anyone with the link can view this page → 任何有链接的人都可以查看此文档
- Make this page publicly accessible → 使此文档可公开访问
- Allow search engines to index page → 允许搜索引擎索引文档
- Publicly shared pages from spaces → 您所在空间的公开共享文档

### 9. 导入导出（5处）
- Import pages → 导入文档
- Import pages & space settings → 导入文档和空间设置
- Importing pages → 正在导入文档
- Page import is in progress → 文档导入正在进行中
- Unable to import pages → 无法导入文档
- Export all pages and attachments → 导出当前空间的所有文档和附件

### 10. 垃圾箱相关（3处）
- Pages in trash will be permanently deleted after 30 days → 垃圾箱中的文档将在30天后被永久删除
- No pages in trash → 垃圾箱中没有文档

### 11. 空间管理（2处）
- Delete this space with all its pages and data → 删除空间及其所有文档和数据
- All pages, comments, attachments and permissions → 此空间中的所有文档、评论、附件和权限

### 12. 评论类型（1处）
- Page (comment type) → 文档

## 保留的翻译

仅保留2处指代网页界面的"页面"：
1. "404 page not found" → "404 页面未找到"
2. "Sorry, we can't find the page you are looking for" → "抱歉，我们无法找到你所需要的页面"

## 修改原则

### 统一性
所有与文档内容相关的术语统一使用"文档"，包括：
- 文档本身
- 文档操作（创建、编辑、删除等）
- 文档属性（历史、宽度等）
- 文档关系（子文档、父文档）
- 文档状态（已删除、已恢复等）

### 准确性
仅在指代网页界面时保留"页面"，确保术语使用的准确性。

### 一致性
确保整个应用中文界面的术语使用完全一致，提升专业性。

## 用户体验提升

### 1. 更符合中文习惯
"文档"比"页面"更贴切地表达用户创建的内容，更符合中文用户的使用习惯。

### 2. 术语统一
整个应用中统一使用"文档"，避免了"页面"一词在不同场景下的歧义。

### 3. 专业性提升
统一的术语使用让应用看起来更加专业和精心设计。

## 总结

本次修改实现了中文翻译的完全统一，将所有与文档内容相关的"页面"改为"文档"，共修改50+处翻译。这不仅提升了用户体验，也使应用看起来更加专业和精心设计。唯一保留"页面"的地方是404错误页面，因为这里指的是网页本身，而非文档内容。

修改后的翻译更符合中文用户的使用习惯，术语使用更加准确和一致，为用户提供了更好的使用体验。
