# 工作空间翻译修正说明

## 问题描述

用户反馈：工作空间设置页面中，图标标签依然显示"站点 icon"，工作空间名称应该显示为"站点名称"。

## 修改内容

### 1. 工作空间图标翻译
- **翻译键**: `"Workspace icon"`
- **修改前**: "工作空间图标"
- **修改后**: "站点图标"

### 2. 工作空间名称翻译
- **翻译键**: `"Workspace Name"`
- **修改前**: "工作空间名称"
- **修改后**: "站点名称"

## 术语体系

修改后的完整术语体系：

```
站点 (Workspace)
  ├── 图标: 站点图标
  ├── 名称: 站点名称
  └── 文档库 (Space)
      ├── 图标: 文档库图标
      ├── 名称: 文档库名称
      └── 文档 (Document)
          └── 子文档 (Subpage)
```

## 翻译对照表

| 上下文 | 英文 | 中文 |
|--------|------|------|
| 工作空间图标 | Workspace icon | 站点图标 |
| 工作空间名称 | Workspace Name | 站点名称 |

## 术语说明

### 为什么使用"站点"？
1. **简洁性**: "站点"比"工作空间"更简洁
2. **通用性**: "站点"是用户熟悉的概念
3. **层级清晰**: "站点 → 文档库 → 文档"的层级关系一目了然

### 术语层级关系
- **站点** (Workspace): 最顶层，整个应用的容器
- **文档库** (Space): 中间层，用于组织文档
- **文档** (Document/Page): 基础单元，实际内容
- **子文档** (Subpage): 文档的子级
