# 空间改为文档库翻译说明

## 修改概述

根据用户需求，将中文翻译中所有的"空间"（Space）统一改为"文档库"，以提供更准确、更符合中文习惯的术语。

## 修改原因

1. **术语准确性**: "文档库"比"空间"更准确地表达了这个功能的本质——存储和管理文档的地方
2. **用户理解**: "文档库"对中文用户来说更直观，更容易理解
3. **术语一致性**: 与"文档"的命名保持一致，形成"文档库 → 文档"的层级关系

## 修改统计

共修改了 **40+** 处翻译，涵盖以下类别：

### 1. 基础概念（7处）
- Space → 文档库
- Spaces → 文档库
- Space name → 文档库名称
- Space description → 文档库描述
- Space settings → 文档库设置
- Space menu → 文档库菜单
- Space slug → 文档库短链接

### 2. 操作相关（6处）
- Create space → 创建文档库
- Delete space → 删除文档库
- Export space → 导出文档库
- View all spaces → 查看所有文档库
- Find a space → 查找文档库
- Search for spaces → 搜索文档库

### 3. 成员管理（3处）
- Add space members → 添加文档库成员
- Remove space member → 移除文档库成员
- Spaces you belong to → 您所属的文档库

### 4. 权限相关（3处）
- Can create and edit pages in space → 能够在文档库中创建和编辑文档
- Can view pages in space but not edit → 能够在文档库中读取文档，但不允许编辑
- Has full access to space settings and pages → 能够更改全部文档库设置和文档

### 5. 提示消息（4处）
- Space created successfully → 文档库创建成功
- Space updated successfully → 文档库更新成功
- Space deleted successfully → 文档库已成功删除
- No space found → 未找到文档库

### 6. 文档操作（4处）
- Move page to a different space → 将文档移动到不同的文档库
- Copy page to a different space → 将文档复制到不同的文档库
- Copy to space → 复制到文档库
- Import pages & space settings → 导入文档和文档库设置

### 7. 确认对话框（4处）
- Are you sure you want to delete this space? → 您确定要删除这个文档库吗？
- Delete this space with all its pages and data → 删除文档库及其所有文档和数据
- All pages, comments, attachments and permissions in this space → 此文档库中的所有文档、评论、附件和权限
- Confirm space name → 确认文档库名称
- Type the space name to confirm → 输入文档库名称以确认

### 8. 其他（5处）
- Export all pages and attachments in this space → 导出当前文档库的所有文档和附件
- Publicly shared pages from spaces you are a member of → 您所在文档库的公开共享文档
- Search in all your spaces → 在您的所有文档库中搜索
- e.g Space for product team → 例如：产品团队的文档库
- Invited members will be granted access to spaces → 被邀请的成员将被授予访问群组可以访问的文档库的权限

## 保留的翻译

保留"工作空间"（Workspace）相关的翻译，因为这是不同的概念：
- Workspace → 工作空间
- Workspace Name → 工作空间名称
- Workspace settings → 工作区设置
- Setup workspace → 设置工作空间
- Join the workspace → 加入工作空间
- Can become members of groups and spaces in workspace → 可以成为工作区中群组和文档库的成员

## 术语层级关系

修改后的术语形成了清晰的层级关系：

```
工作空间 (Workspace)
  └── 文档库 (Space)
      └── 文档 (Page/Document)
          └── 子文档 (Subpage)
```

这样的层级关系更加清晰，用户更容易理解：
- **工作空间**: 最顶层，包含多个文档库
- **文档库**: 中间层，用于组织和管理文档
- **文档**: 基础单元，实际的内容
- **子文档**: 文档的子级
